Fullstack dev is boring

May 14, 2026

Hear me out, it's a me problem

I'm currently an incoming junior, and I haven't built much projects. To be fair, I'm too busy serving as a lead dev for my campus's hackathon group this summer as well as taking the role of technical lead for Google Developer Groups on Campus at UM-Dearborn, so I haven't had time.

Honestly, I don't think I built enough yet to the point where I explored plenty of various possibilities of what I want to do in life. But everytime I try building a new project, I feel like it's the same thing over and over again. My anxiety of building repetitive projects has killed my learning curve, and won't help me reach beyond what I truly want to still learn.

The shift

To help me learn beyond what I know, I plan on changing what I build. Although it's all talk, I just want to remind myself that I'm not a beginner anymore. I'm not making basic projects lol.

  • LLM APIs ❌ → AI Agent SDKs/Frameworks ✅
  • Web dev ❌ → Mobile and IoT dev ✅
  • Mock data ❌ → Real world data ✅
  • Manually-built components ❌ → 3rd party integrations ✅
  • Manually-built auth ❌ → 3rd party auth ✅
  • Basic frontend/backend/database/LLM API stack ❌ → integrating CV, ML, DL, etc. ✅

Hopefully, these are things that i want to change when building new projects

Things I currently want to learn through building

  • IoT dev
  • AI Agent concepts
  • Computer Vision
  • Machine Learning
  • Deep Learning
  • Virtual Language Models
  • Google Cloud
  • Something with C++ like CUDA?
  • Low-level dev like on the operating system level using Rust or C++

Everything ties back to AI, so maybe I want to build an agentic terminal inspired by Warp for example and how it works under the hood. I have to remember I am hungry to learn about AI.

What stays the same

  • Still using Python, TypeScript, Javascript
  • Still using Next.js, React, FastAPI, MongoDB, and Supabase
  • Still using Google Gemini API and other LLM APIs

In short, my old skills that are transferrable. Now that I'm good at SWE at the bare minimum imo thanks to building so much fullstack projects through vibe-coding, I need to focus more on the big boi stuff. It's not just calling an abstract LLM API like OpenAI API and claiming you're an AI engineer with a depth of knowledge. That's not gonna cut it.

I'm glad I realized before I graduate that I'm learning nothing if I keep building the same thing over and over again disguised as different projects.

Also, my leadership experience on campus compensates for the time where I could've built projects on my own, so I don't think I need to build anymore fullstack projects on my free time like old times sake 😭
